# Peekaboo-Style Capability Parity Matrix (March 4, 2026)

Last updated: March 4, 2026

## Scope

This matrix is evidence-only. Every status below is grounded in concrete code/docs files from:

- Looksy (`protocol/`, `host/`, `tests/`)
- OpenClaw (`../openclaw/src/gateway/server-methods/browser.ts` + routing tests)
- Trope (`../trope/packages/rust/trope-daemon/src/tools/mod.rs`, `../trope/apps/windows/WindowsAgent/Program.cs`)

Status legend used in cells:

- `Implemented`: command/capability exists and is wired.
- `Mapped`: consumer route/action translation exists.
- `Simulated`: behavior is in-memory/mock-style, not real platform/browser execution.
- `Missing`: no concrete command/mapping implementation found.

## Highest-Priority Blockers (P0)

1. Command-type divergence for clipped screenshots (`screen.capture.region` vs Looksy `screen.capture` command list).
2. Browser-state families (`navigate`, `snapshot`, `pdf`, `console`, `trace`) are not real-browser backends in Looksy Windows adapter.
3. Trope Windows runtime `automation.browser` handles OS-input/screenshot subset only; non-covered browser actions fall to `unsupported_method`.
4. Looksy element family is protocol-defined but currently simulated in Windows adapter and unmapped in OpenClaw/Trope Looksy paths.
5. Drag/swipe commands are absent from Looksy protocol and both consumer Looksy mappings.
