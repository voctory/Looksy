## Scope

This matrix is evidence-only. Statuses below are tied to concrete files in:

- Looksy (`protocol/`, `host/`, `tests/`)

- Trope (`../trope/packages/rust/trope-daemon/src/tools/mod.rs`, `../trope/apps/windows/WindowsAgent/Program.cs`)

## Highest-Priority Blockers (P0)

1. OpenClaw clipped screenshot command mismatch (`screen.capture.region` vs Looksy `screen.capture`).
2. OpenClaw still blocks `act:drag`/`act:swipe`/clipboard flows despite protocol + Windows backend support.
3. Trope Looksy mapper still rejects drag/swipe/clipboard and does not map browser PDF/console.
4. Browser-state families remain simulated in Looksy Windows backend, while Trope Windows runtime supports only an OS-input subset.
5. Element family is not wired through OpenClaw/Trope Looksy mappings and still falls back to simulated data in Looksy when UIA lookup fails.
